# Term Deposit Classification

<a target="_blank" href="https://cookiecutter-data-science.drivendata.org/">

End-to-end machine learning pipeline for binary classification of term-deposit
subscriptions using the Bank Marketing dataset.

The project covers:
- Data ingestion, train/test split, and domain-specific cleaning.
- Feature analysis and transformation utilities.
- Model training with imbalanced-learn + XGBoost.
- Evaluation and reporting artifacts.

## Quick Start

1. Create and activate a Python 3.12 virtual environment.
2. Install dependencies:

```bash
pip install -r requirements.txt
```

3. From here, pick one of two paths:

   - **Fastest — just try the demo:** `models/model_pipeline.joblib` is
     already trained and committed, so you can skip straight to the
     [Streamlit Demo App](#streamlit-demo-app) below without running
     anything else.
   - **Full pipeline reproduction** (re-cleans the data and retrains the
     model from scratch):

     ```bash
     python deposit_classification/dataset.py
     python deposit_classification/plots.py
     python deposit_classification/features.py
     python deposit_classification/modeling/train.py
     python deposit_classification/modeling/predict.py
     ```

## Data And Outputs

- Input CSV: `data/raw/term-deposit-marketing-2020.csv`
- Train/test pickles:
    - `data/processed/X_train.pkl`
    - `data/processed/y_train.pkl`
    - `data/processed/X_test.pkl`
    - `data/processed/y_test.pkl`
- Trained model: `models/model_pipeline.joblib`
- EDA figures: `reports/figures/` (generated by `deposit_classification/plots.py`)
- Analysis tables: printed to terminal by `deposit_classification/plots.py` and
    `deposit_classification/features.py`

## Streamlit Demo App

An interactive showcase lives in `streamlit_app.py` at the project root, with
four tabs:

- **Overview** — a walkthrough of the pipeline stages below.
- **Explore the Data** — sample rows, class balance, an interactive
  subscription-rate-by-feature chart, and the pre-generated EDA figures.
- **Try a Prediction** — fill in a customer profile and get a live
  subscription probability from the trained pipeline.
- **Model Performance** — accuracy, precision/recall, F1, ROC AUC, a
  confusion matrix, and an ROC curve on a held-out test split.

It only needs `models/model_pipeline.joblib` (already committed to this repo)
to serve predictions, so it works right after `pip install -r
requirements.txt` with no other setup. If `data/raw/term-deposit-marketing-2020.csv`
is also present locally, the Explore Data and Model Performance tabs are
enabled too — that CSV is gitignored, so it won't be there on a fresh clone
from GitHub unless you add it yourself.

Run it with:

```bash
make app
```

or directly, from either the project root or its parent directory:

```bash
streamlit run streamlit_app.py
```

│   ├── interim        <- Intermediate data that has been transformed.

│   └── raw            <- The original, immutable data dump.
│
├── docs               <- MkDocs project for user and developer documentation
│
├── models             <- Trained and serialized models, model predictions, or model summaries
│
├── notebooks          <- Jupyter notebooks. Naming convention is a number (for ordering),

│                         `1.0-jqp-initial-data-exploration`.
│
├── pyproject.toml     <- Project configuration file with package metadata for 
│                         deposit_classification and configuration for tools like black
│
├── references         <- Data dictionaries, manuals, and all other explanatory materials.
│
├── reports            <- Generated analysis as HTML, PDF, LaTeX, etc.
│   └── figures        <- Generated graphics and figures to be used in reporting
│
├── requirements.txt   <- Project dependencies for reproducible local runs
│
├── streamlit_app.py   <- Interactive Streamlit demo app (see below)

└── deposit_classification   <- Source code for use in this project.
    │
    ├── __init__.py             <- Makes deposit_classification a Python module
    │
    ├── config.py               <- Store useful variables and configuration
    │
    ├── dataset.py              <- Scripts to download or generate data
    │
    ├── features.py             <- Code to create features for modeling
    │
    ├── modeling

    │   └── train.py            <- Code to train models
    │
    └── plots.py                <- Code to create visualizations
